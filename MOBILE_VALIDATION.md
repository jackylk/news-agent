# 手机端访问验证清单

## 部署完成后，请按以下步骤验证手机端访问

### 1. 获取 Vercel 部署域名

部署完成后，从 Vercel 控制台获取部署域名，例如：
- `https://news-agent-xxx.vercel.app`
- 或你的自定义域名

### 2. 电脑端验证（先验证基本功能）

在电脑浏览器中打开 Vercel 域名，验证：
- [ ] 页面正常加载
- [ ] 新闻列表显示正常
- [ ] 可以点击新闻查看详情
- [ ] 分类筛选功能正常
- [ ] 翻译功能正常（英文文章）
- [ ] 打开浏览器开发者工具（F12），检查：
  - [ ] Network 标签中所有 API 请求都成功（状态码 200）
  - [ ] 没有 CORS 错误
  - [ ] 没有混合内容警告（所有请求都是 HTTPS）

### 3. 手机端验证

#### Android Chrome 浏览器

1. 在手机 Chrome 浏览器中打开 Vercel 域名
2. 验证以下功能：
   - [ ] 页面正常加载（不是空白页）
   - [ ] 新闻列表显示正常
   - [ ] 可以滚动浏览新闻列表
   - [ ] 点击新闻条目可以打开详情页
   - [ ] 详情页内容正常显示
   - [ ] 可以点击"返回列表"按钮返回
   - [ ] 分类筛选功能正常（点击分类标签）
   - [ ] 新闻源筛选功能正常（点击新闻源标签）
   - [ ] 如果是英文文章，翻译功能正常

3. 如果遇到问题，打开 Chrome 的开发者工具：
   - 在 Chrome 地址栏输入 `chrome://inspect`
   - 连接手机后，点击 "inspect" 查看控制台错误

#### iOS Safari 浏览器

1. 在 iPhone Safari 浏览器中打开 Vercel 域名
2. 验证功能（同 Android Chrome）
3. 如果遇到问题：
   - 在 Mac 上打开 Safari
   - 菜单栏：开发 > [你的 iPhone] > [网页]
   - 查看控制台错误

### 4. 常见问题排查

#### 问题：页面空白

**可能原因：**
- API 请求失败
- JavaScript 错误

**解决方法：**
1. 打开手机浏览器开发者工具（或远程调试）
2. 查看控制台错误信息
3. 检查 Network 标签，确认 API 请求是否成功
4. 确认 Railway 后端服务正常运行

#### 问题：API 请求失败

**可能原因：**
- CORS 配置问题
- Railway 服务未运行
- 网络连接问题

**解决方法：**
1. 检查后端 CORS 配置（`server/server.js` 中 `origin: '*'` 应该允许所有来源）
2. 确认 Railway 服务正常运行
3. 在电脑浏览器中测试相同功能，确认是否是手机端特定问题

#### 问题：页面显示但功能不正常

**可能原因：**
- JavaScript 执行错误
- 移动端兼容性问题

**解决方法：**
1. 查看浏览器控制台错误
2. 检查是否有 JavaScript 语法错误
3. 确认移动端浏览器版本是否过旧

### 5. 验证清单总结

完成所有验证后，确认：
- [ ] 电脑端所有功能正常
- [ ] 手机端（Android Chrome）所有功能正常
- [ ] 手机端（iOS Safari，如果有 iPhone）所有功能正常
- [ ] 没有控制台错误
- [ ] 没有网络请求错误
- [ ] 用户体验流畅

## 如果验证失败

如果手机端仍然无法访问或功能不正常，请：
1. 记录具体的错误信息（截图或文字描述）
2. 检查浏览器控制台错误
3. 检查 Network 标签中的请求状态
4. 确认 Vercel 部署日志是否有错误
5. 确认 Railway 后端服务是否正常运行
